Most Common Git Commands

The first thing we need to do, is to check if Git is properly installed:

      git --version

Configure Git

Note: Use global to set the username and e-mail for every repository on your computer. If you want to set the username/e-mail for just the current repo, you can remove global

      git config --global user.email "you@example.com"
      git config --global user.name "Your Name"
    

      git config --list

Initialize Git

Initialize Git on that folder:

      git --init

Git Adding New Files

README.md file that describes the repository (recommended for all repositories):

List the files in our current working directory:

      ls

Then we check the Git status and see if it is a part of our repo:

      git status
      git status --short

Note: Short status flags are:

  • ?? - Untracked files
  • A - Files added to stage
  • M - Modified files
  • D - Deleted files

Git Add

      git add <file_name>
      git add .
      git add --all (or) git add -A
      git add *<files_type>

Git Reset

Undo a git add

      git reset <file_name>

Git Commit

Since we have finished our work, we are ready move from stage to commit for our repo.
When we commit, we should always include a message. 

      git commit -m "message"

Git Commit Log

To view the history of commits for a repository, you can use the log command:

      git log

Git Help

There are a couple of different ways you can use the help command in command line:

  • git command -help - See all the available options for the specific command
  • git help --all - See all possible commands
      git command -help
      git help --all

Git Branch

In Git, a branch is a new/separate version of the main repository.

Create a new branch:

      git branch <branch_name>

List the existing branches:

      git branch

Delete the branch, as it is no longer needed:

      git branch -d <branch_name>

Moving us from the current branch, to the one specified at the end of the command:

      git checkout <branch_name>

Emergency Branch

Create a new branch to deal with the emergency:

Create, and move to a new branch with the branch_name:

      git checkout -b emergency-fix

Git Branch Merge

Merge from the branch_name with the current branch:

      git merge <branch_name>

Remove the branch_name from the local repository:

      git branch -d <branch_name>

Git Remote

Add a remote repository as an origin:

      git add origin https://github.com/username/repository-name.git

Git Pull from Remote

pull is a combination of: fetch and then merge

Get all the change history of the origin for this branch:

      git fetch origin

Merge the current branch with the branch master, on origin:

      git merge origin/master

Update the current branch from its origin using a single command:

      git pull origin

Git Push to Romote

push the current branch to its default remote origin:

      git push origin

Git Pull Branch from Romote

List all local and remote branches of the current Git.

      git branch -a

List only remote branches of the current Git.

      git branch -r

Git Clone

Clone the repository to your local Git:

      git clone https://github.com/username/repository-name.git

Clone the repository github to a folder named "new-folder":

      git clone https://github.com/username/repository-name.git "new-folder"

Rename the origin remote to upstream:

      git remote rename origin upstream

Git Revert

Show the log of the repository, showing just 1 line per commit:

      git log --oneline

revert the latest commit:

      git revert HEAD

revert the latest commit, skipping the commit message editor:

      git revert HEAD --no-edit

revert the two last commits:

      git revert HEAD~1

Git Reset

reset to the commit with the hash:

      git reset e214sd

Amend the previous commit to with the message "Updated new message":

      git commit --amend "Updated new message"
